Windows和IOS同步Obsidian

目标

  1. 以PC编辑操作为主，手机端只是用来查看
  2. 保留手机端一个目录，这个目录下可以新增编辑文件，可以传回PC端

方案

  1. 安装使用 Remotely Save，最新版的选项比之前版本更加丰富，可以指定同步目录，有四种同步方式
  2. PC端、手机端的配置文件分开，PC端默认为 .obsidian，手机端默认为 .obsidian-mobile，可以在PC端、手机端分别配置。设置 - 文件与链接 - 切换配置文件夹
  3. 手机端保留一个目录 IOS，设置默认在这个目录中新建文件 设置 - 文件与链接 - 新建笔记的存放位置 和 设置 - 文件与链接 - 存放新建笔记的文件夹
  4. PC端 Remotely Save 配置
    1. 选择远程服务，选择合适的方式，我原来用 onedrive 同步，现在自己搭了一个 webdav 服务
    2. 同步方向，选择 增加推送带删除，由PC单向同步到服务器
    3. 忽略的文件或文件夹的正则表达式，选择 IOS 目录，不同步这个目录，移动端配置双向同步，这样移动端这个目录下的改动可以同步回PC端，而其它目录的内容则单向从PC端同步到移动端
    4. 其它的改动，包括 同步_文件或文件夹，同步配置文件夹，同步书签，根据自己的需要全部打开
  5. 移动端 Remotely Save 配置
    1. 选择远程服务，配置同PC端
    2. 同步方向，选择双方同步。这里如果选择 增量拉取带删除，会造成 IOS 目录下的改动无法同步到服务端，另外所有操作动作记录都是配置文件中，配置文件也会被覆盖掉
    3. 忽略的文件或文件夹的正则表达式 为空
    4. 其它的改动，包括 同步_文件或文件夹，同步配置文件夹，同步书签，根据自己的需要全部打开
  6. 服务器上的 IOS 目录同步回PC
    1. 使用任何一个同步软件同步即可，这里使用 syncback pro。如果 Remotely Save 使用 onedrive 同步方案，还可以将 onedrvie 服务端 IOS 目录软连接到OBS的vault中，直接使用
